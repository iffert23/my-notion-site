온-프레미스 애플리케이션에 액세스하는 원격 사용자를 인증하려면 Microsoft Entra ID를 사용해야 합니다.

설계에 무엇을 포함해야 합니까?

답변을 하나만 선택하세요.

- [ ]  Microsoft Entra Domain Services
- [ ]  Microsoft Entra 애플리케이션 프록시
- [ ]  Microsoft Entra Connect
- [ ]  비즈니스용 Windows Hello

<aside>
💡

애플리케이션 프록시를 사용하면 온-프레미스 애플리케이션이 Microsoft Entra ID의 토큰을 사용할 수 있습니다. 비즈니스용 Windows Hello는 Windows 클라이언트의 MFA(다단계 인증)에 사용됩니다. Microsoft Entra Connect는 암호 해시를 동기화하는 데 사용됩니다. Microsoft Entra DS는 클라우드의 관리 Active Directory 서비스입니다.

[ID 및 IAM(액세스 관리)을 위한 디자인 - 학습 | Microsoft Learn](https://learn.microsoft.com/training/modules/design-authentication-authorization-solutions/2-design-for-identity-access-management)

VNet1 Azure Virtual Network가 있습니다.

VNet1 진단 데이터를 저장할 계획입니다.

다음 요구 사항을 충족하는 진단 데이터 저장 솔루션을 권장해야 합니다.

- 모든 로그에 대한 고속 액세스 제공
- 모든 로그를 5년 동안 보관합니다.
- 로그 항목을 수정할 수 없도록 보장합니다.
- 5년 후 모든 로그가 자동으로 제거되도록 보장합니다.

권장 사항에 포함해야 하는 두 가지 옵션은 무엇인가요? 각 정답은 해답의 일부를 나타냅니다.

해당하는 답변을 모두 선택하세요.

- [ ]  변경이 불가능한 스토리지 정책
- [ ]  Premium Blob Storage
- [ ]  보관 액세스 계층
- [ ]  쿨 액세스 계층

<aside>
💡

변경할 수 없는 스토리지 법적 보존 및 Premium Blob Storage는 로그가 필요한 시간 동안 유지되고 변경 불가능한 상태로 저장되도록 합니다. 즉, 저장한 후에는 변경할 수 없습니다.

모든 로그에 대한 고속 액세스가 필요한 경우 보관 및 쿨 액세스 계층은 적합하지 않습니다.

조직에서 온-프레미스 인프라를 Microsoft Azure로 마이그레이션하여 회사 거버넌스 정책을 준수해야 합니다.

규정 준수 검사를 자동화하고 Azure 리소스에 정책을 적용하는 전략을 설계해야 합니다.

각 정답은 해답의 일부를 나타냅니다. 수행해야 하는 세 가지 작업은 무엇입니까?

해당하는 답변을 모두 선택하세요.

- [ ]  관리 그룹 수준에서 정책을 할당합니다.
- [ ]  Microsoft Azure Security Center를 배포합니다.
- [ ]  Microsoft Azure Policy를 구현합니다.
- [ ]  Azure Blueprints를 사용하여 규정 준수 표준을 정의하고 배포합니다.
- [ ]  규정 준수 자동화에 Azure Logic Apps를 사용합니다.
- [ ]  Microsoft Azure Cost Management를 사용합니다.

<aside>
💡

관리 그룹 수준에서 정책을 할당하면 구독 간에 일관된 적용이 보장되며 이는 회사 거버넌스 정책 준수를 유지하는 데 매우 중요합니다. Microsoft Azure Policy를 구현하면 리소스에 대한 규칙 및 효과를 적용하여 규정 준수 검사를 자동화하여 기업 거버넌스 정책을 준수할 수 있습니다. Microsoft Azure Cost Management는 규정 준수 유지 관리의 핵심 구성 요소인 거버넌스의 비용 관리 측면에 맞춰 지출을 추적하고 제어하는 데 도움이 됩니다. 

Microsoft Azure Security Center는 이 시나리오에 필요한 거버넌스 정책 적용 또는 규정 준수 자동화를 구체적으로 다루지 않는 보안 상태 관리 및 위협 방지에 중점을 둡니다. Azure Blueprints를 사용하면 규정 준수 표준을 정의하고 배포하는 데 도움이 되지만 이 시나리오에 필요한 지속적인 규정 준수 검사 또는 적용을 자동화하지는 않습니다. Azure Logic Apps는 워크플로를 자동화하기 위한 강력한 도구이지만 이 시나리오에 필요한 거버넌스 정책을 적용하거나 규정 준수 검사를 자동화하도록 설계되지 않았습니다.

[Azure 랜딩 존의 디자인 영역 - 교육 | Microsoft Learn](https://learn.microsoft.com/training/modules/cloud-adoption-framework-ready/4-design-areas)

[클라우드 거버넌스 정책 적용 - 교육 | Microsoft Learn](https://learn.microsoft.com/training/modules/cloud-adoption-framework-govern/6-enforce-policies)

[Azure 거버넌스 디자인 영역 - 클라우드 채택 프레임워크 | Microsoft Learn](https://learn.microsoft.com/azure/cloud-adoption-framework/ready/landing-zone/design-area/governance)

</aside>

Azure Container Instances와 같은 Azure Container Apps에는 기본 제공 부하 분산 및 관리형 TLS가 부족하지만 Azure Functions는 컨테이너별 요구 사항을 충족하지 않는 서버리스 솔루션입니다.
