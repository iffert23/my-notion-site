# NAC란? (+NAC의 필요성)

---

- **참고**
    
    **2022년 파트너 NAC5.0 초급교육(1. 제품소개)**
    
- Network Access Control
- 컴퓨터-컴퓨터가 물리/논리적으로 연결되어 있는 망에 접근하는 것을 제어,통제하는 것
- NAC 제품은 네트워크 접근 통제 솔루션

**NAC의 필요성**

- 가시성
    - IT환경이  PC → BYOD로 변화 ⇒ 내부 네트워크자원 사용장치 종류 증가 ⇒ 내부 네트워크 사용 단말의 종류, 상태, 현황, 사용자에 대한 가시성 확보가 중요함
    - 네트워크에 연결된 디바이스가 늘어날수록 보안관리자의 부담도 증가 ⇒ 가시성 확보가 매우 중요, 기본적인 사항임
- 지니안 NAC 5.0
    
    내부 정보보호체계 수립, 내부 자산&사용자 보호, 기업자원 안전사용을 지원하는 유무선 네트워크 접근제어 솔루션
    
    - 강력한 인증 → 자산/사용자 식별, 네트워크 접근권한 차등 부여, 사용자 접속권한 제어(IP/SUBNET/VLAN), 사용자 단말 보안상태 점검&조치
    
    ⇒ 유/무선을 아우르는 내부 보안관리체계 완성
    
- 네트워크 가시성
    - 네트워크 자원을 공유하며 내부 네트워크에 엑세스하는 다양한 종류의 단말 중 어떤 사용자가 어떤 단말을 사용하는지에 대한 인지가 필요함
    - 누가 언제 어느 곳을 어떤 단말로 접근했는지에 대한 관리권한이 필요하다
- End-Point 가시성
    - 내부 네트워크 사용단말의 상태관리가 없으면 내부 네트워크의 안전성을 책임질 수 없는 보안환경
    - 악성코드에 감염된 단말인지
    - 단말에 최신 보안패치가 있는지
    - 유선/무선 중 무엇을 사용중인지
    - 사내 규정에 맞는 소프트웨어를 사용중인지
    - 테더링 사용중인지
    - 단말 운영체제가 뭔지
- 사용자 가시성
    - 누가 언제 어떤 기기를 사용해서 내부 네트워크를 사용했는지 감사기록(증적) 필요
    - 내부 인프라접속 절차확인&감사증적(로그기록)
- 보안규정 시행강화
    - 업종별 내부조안 규정시행 → 기업(관)의 내부보안에 대한 적절한 보호조치 요구
- ISMS 인증
    - 2016 → 인증 의무대상 확대, 위반 시 과태료가 3,000만원으로 상향조정 됨
    - 인증항목 중 NAC와 직접 관련된 내용

    - Full cycle 보안정책 수립&시행 필요
    - 사용자/단말/기기 등의 다양한 조건을 통한 누락없는  보안정책 적용
