언더레이 : 물리적인 네트워크 인프라를 구성하는 것. 물리적이라는건 네트워크 장비들끼리 직접 연결해서 구성하는 인프라를 말함. 단점으로는 낮은 보안성이 있음. 트래픽이 평문통신으로 이루어진다.

인 줄 알았는데 조금 애매해서 이것저것 검색해봄

언더레이=데이터 플레인 : 네트워크에서 실제 네트워크가 전송되는 경로(?)

오버레이 : 물리적인 인프라를 기반으로 한 네트워크 가상화 기술임. 터널구성하는것

종류에는 VxLAN, IPSec, MLPS, EVPN, GRE, SDN 있음

데이터 전송을 가상화된 터널을 통해 전송하기 때문에 전송속도가 빠르다. 가상화기술이기 때문에 확장성이 좋다. 여러가지 프로토콜을 사용할 수 있음.

오버레이=컨트롤플레인 : 네트워크 장비들끼리 서로 상호작용하고 제어하는 논리적인 경로 인가

오리지널 헤더가 의미 있는 영역, 터널 헤더가 의미 있는 영역.

Best-Path를 찾는 것은 오리지널 헤더로 하는거임→오버레이

추가적인 헤더(터널헤더)를 기반으로 Best-Path를 찾고 필터링을 하고 QoS를 하는 것→언더레이

터널헤더가 오리지널헤더 하위레이어에 존재, 레벨이 높은 건 알 수 없음

IPSec : 인터넷 프로토콜 보호하는 것. 네트워크상에서의 안전한 연결을 위한 통신 프로토콜 묶음

오버레이 방식은 셀룰러 통신 링크와 D2D 통신 링크 간 직교하게 자원을 할당하는 방식이고, 언더레이 방식은 셀룰러 통신 링크와 D2D 통신 링크 간 자원을 공유하는 방식이다.

GRE : 일반 라우팅 켑슐화.

라우팅이 불가능한 패킷을 라우팅 가능한 패킷의 내부에 넣어서 전송할 때 사용하는 터널링 프로토콜이다, 근데 라우팅이 불가능한 패킷이 뭔디..?  

→ 이해함. 내가 있는 A에서 상대편 B로 패킷을 보내야 하는데 A에서 B로 갈 때 거쳐야 하는 네트워크에서 우리쪽 프로토콜을 지원하지 않는 상황임. 그러면 내가 가진 패킷을 거쳐야 하는 네트워크에 호환되는 패킷 안에 캡슐화해서 트래픽을 보내면 되는 것.
