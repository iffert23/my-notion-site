# 2장/ 네트워크 연결과 구성요소

---

### 2.1 네트워크 연결 구분

- LAN (Local Area Network)
    
    비교적 소규모의 네트워크
    
    자신이 소유한 건물이나 대지에 직접 구축한 선로로 동작시키는 네트워크 (복잡하거나 대규모여도 직접 구축한 범위라면 LAN임)
    
- MAN
- WAN
    
    멀리 떨어진 LAN을 서로 연결/인터넷에 접속하도록 하는 네트워크
    
    통신사업자로부터 회선 임대 (원격지로 통신해야 할 때 사용)
    

### 2.2 네트워크 회선

- 인터넷 회선
    
    인터넷 접속을 위해 통신사업자와 연결하는 회선
    
    → 통신 사업자가 판매하는 인터넷과 연결된 회선을 사용해야 함
    
    → 전송 선로 공유기술 사용 (ex. 아파트: 아파트에서 통신사업자까지 연결한 회선을 아파트 가입자들끼리 공유→주변 사용량에 따라 속도가 느려질 수 있음)
    
    ![Untitled](2%EC%9E%A5%20%EB%84%A4%ED%8A%B8%EC%9B%8C%ED%81%AC%20%EC%97%B0%EA%B2%B0%EA%B3%BC%20%EA%B5%AC%EC%84%B1%EC%9A%94%EC%86%8C/Untitled.png)
    
- 전용 회선
가입자와 통신사업자 간에 대역폭을 보장해주는 서비스 →전용케이블 연결
통신사업자 내부에서 TDM같은 기술로 직접 연결한 것처럼 품질 보장
    - 저속 : 음성 전송 기술 기반
    작은 기본 단위를 묶어 회선 접속 속도를 높이는 방법(64kbps단위)
    과거→높은 속도 필요X/높은 신뢰성 필요할 때 사용
    현재→사용 빈도 줄고있지만 Clear Text(전문전송, 결재승인)을 위한 VAN사나 대외 연결에는 사용하는 경우 많음
    ⇒ 원격지 전송 기술로 변환할 수 있는 라우터 필요
    - 고속 : 메트로 이더넷
    광케이블 기반 이더넷 사용
    가입자-통신사업자 접속기술은 이더넷
    통신사업자 내부에서는 개별 가입자 묶어서 통신 가능한 다른 고속 통신 기능 사용
    - (추가)LLCF
    한쪽 링크가 다운되면 이를 감지해 반대쪽 링크도 다운시켜서 장애를 감지 할 수 있게 함
- 인터넷 전용 회선
인터넷 연결 회선에 대한 통신 대역폭을 보장해줌으로써 다른 가입자와 경쟁없이 연결 품질 보장
가입자-통신사업자 → 인터넷 연결구조
- VPN
Virtual Private Network
전용선이 아니지만 가상으로 직접 연결한 것 같은 효과가 나도록 만들어줌
    - 통신사업자 VPN
    비싼 전용선 대역폭 낭비 예방을 위해서 통신사업자가 직접 가입자를 구분할 수 있는 VPN
    대표적으로 MPLS VPN이 있음
    - 가입자 VPN
    일반 사용자가 사용하는 VPN. 
    일반 인터넷망을 이용해 사용자가 직접 가상 전용 네트워크 구성 가능
- DWDM (파장 분할 다중화)
    - 먼 거리를 통신할 때 케이블 포설 비용이 많이 들고 어쩌고 단점 극복위해 개발
    - 하나의 광케이블에 다른 파장의 빛을 통해 여러 채널을 만들며 많은 데이터를 전송할 수 있음
    - FTTH 구축방식 구분
        - PTP : 가입자-통신사업자 케이블 직접 포설
        - AON : 광신호 분리장비에 전기가 필요한 스위치같은 장비 사용
        - PON : 전기 인입 없이 광신호 분리, 가입자-통신사업자 케이블 줄일수있음 → 속도 느려질 수 있음 → DWDM 접목
