# 네트워크 구성 변경 미팅

상태: 완료
날짜: 2025년 9월 9일

김준겸프로 : SDS 사람 필요한 말만 하면 됨, 말이 좀 센 편이라고 함

우리쪽에서는 대역 인바운드 아웃바운드 필터 걸 수 있다. 그리고 추가적으로 나중에 이진팀장님이 자세히 말씀드릴건데 2월말쯤 되면 우리쪽에서 필터를 거는 게 아니라 온프렘에서 아예 필요 없는 대역은 좀 솎아내고 제거하는 과정을 거칠 필요가 있어보인다 정도만 말하면 되고

오늘 회의때 윤상문 프로가 아마 주로 얘기할 것 같다고 함

---

화상회의 DC에서 애저 통해서 가는지 아니면 SDSA로 바로 가는지 선택을 해야한다

105 아이피 그대로 쓸건지? → 105 쓰면 수원DC에서 라우팅이 굉장히 복잡해질것. 그러니까 변경할 수 잇으면 변경해달라. 수원으로 올때만 SCTA 아이피를 NAT로 변경하면 좋음

거점끼리는 IP 변경 없이 수원으로 오는 경우만 IP 변경햇으면 좋겟다

수원은 그룹망 관계 통신이 있을 수 잇음. 그룹망 구간에 접속하려면 수원DC가 SDSA로 가는걸 전용회선으로 던져버린다. 그래서 내부IP로 연결해야함

거점에서 화상회의 장비 쓸 때 클래스 하나 주고 IP 쪼개고 1ㄷ1매핑해서 수원에서 들어올때 90, 요청할때도 90으로 NAT하는 방법 있다

→ 90을 달고 들어왔다가 나갈 수 있나? → 

동서남아처럼 IP를 바꾸되 화상회의만 90으로 바꿔서~

STCA에 있는 화상회의 장비들은 SDSA쪽에 있는 다른 부문이랑 통신하게 된다면 어떻게 되는지? → 그러면 90아이피 갖고 있으면 안됨

SDS쪽에서 90대역을 꺾어주면 안되나? → 되긴 하는데 SDSA 담당자가 안해주여고 할 듯

→ 그런 케이스는 많지 않을듯

전용회선 해지했을때 수원 DC에서 어떻게 할 수 있는지에 대해~~얘기함

→노트북 붙여서 테스트 해볼까?

장비 미국 관세청 통과하다가 잡혔다고함^^;;

---

뉴저지 ipsec 끊어졌을때 휴스턴쪽 또다른 서브 만들어서 SDSA로 붙일거임

휴스턴은 장비 하나만 줄거임 뉴저지는 장비 두개를 주고 AC-ST로 이중화

위 구성을 참고

1. 여기서 제일 중요한건 화상회의인데 화상회의를 삼성본관↔SDSA로 양방향 통신이 되도록 설정해야 함
2. 현재 KR POC 설정 기준 화상회의 IP가 105대역이 될 텐데 구축도 그렇게 되면 수원 DC에서 설정이 복잡해질 우려가 크기 때문에 화상회의 IP를 90대역으로 바꾸는 부분을 추가 논의할 필요가 있다.
3. 2번 방법으로 할 때 IP 자체를 90으로 바꾸는게 아니라 수원DC와 통신 시에만 90대역을 타고 갈 수 있게 NAT 설정을 하는 방법이 제일 효율적일 것으로 생각된다.
    
    ⇒ 거점에서 화상회의 장비 쓸 때 C클래스(/24처럼) 하나 주고 IP 쪼개고 1:1매핑해서 수원에서 들어올때 90, 요청할때도 90으로 NAT하는 방법
    
4. 뉴저지 ipsec 끊어졌을때 휴스턴쪽(백업라인)으로 연결될 수 있도록 또 다른 서브관문을 만들어서 SDSA로 붙일것
    
    ⇒ 휴스턴은 장비 하나(wan1개)만 줄거고 뉴저지는 장비 두개(wan 2개)를 주고 AC-ST로 이중화
