현장작업하면서 테스트병행

대역 미리 받아둬야함. 105대역이랑 미주 녹스 대역이 수원IDC에서 bgp 애드버타이즈닝 되는거 떼는거임. 근데 롱기스트 그거 있어서 

어떻게적용되는지랑 뉴저지 어케되는지가 헷갈림 → 버지니아(이스트유에스)에 허브 만들어야함

허브 만들어두는건 미리 가능함. →의미있냐? 가서 장비를 구성해야대는데 둘 다 한꺼번에 되는건 아니니까 어떤식으로 만드는게 좋을지? ⇒ 전반적인건 출장 현지 일정에 애저 작업을 맞추겠다. 

영향도 두개로 나눠서 보면, 뉴저지 미리 해놓고 휴스턴을 나중에 하면 된다고 했을 때 그 사이에 중남미 다 적용됏을때 다 뉴저지로 가고 백업은 없는 상태에서 중남미 거점은 다 만들고 나중에 휴스턴 만들면 중남미 거점에 뭘 추가해야하는지? 그러면 잘 되는지 아닌지 테스트를 해야되는데 순서를 어떻게 하느냐…

1. 뉴저지 거점 만들고 모든 리전 다 만들고 연결하고 휴스턴 만든다음에 테스트
2. 하나하나 다 하면서 테스트

리소스 배포랑 리스크 발라내기 정의하기 등등은 신경 많이 쓸 거고 어차피 우리는 원격이라 시간 별로 안들어감. 그래서 현장 지원 일정에 다 맞추겠다는것.

fm대로 보면 휴스턴도 같이하는게 맞는것. 여러가지 정황상 휴스턴 안된다고 해도 1번처럼 하는게 좋은 방법임. 휴스턴 다 만든다음에 테스트할때 각 현장이 미주녹스나 그런쪽으로 통신 안되는 시점 확인해서 뉴저지 절체한다음에 페일오버 잘 되는지 테스트해봐야 할 필요성은 있다.

휴스턴이랑 오스틴 일정이 뒤로 밀린다고 하면 그 두 리전은 일단 지금 방식대로 전용선 통신 SDS A

동부지역 리소스 배포. 한국에 잇는 상사 본사 안에 이쓴ㄴ 포티를 테스트장비로 해서 그걸 하나의 온프렘 리전으로 해서 

뉴저지 준비 되면 이스트유에스에 대한 허브랑 게이트웨이 빠르게 만들고, 23비트대역 요청해서 받아서 만든다음에

상사 본사 포티장비로 이스트유에스 게이트웨이랑 ipsec 맺어서 통신 잘 되는지 확인

클라이언트가 밑에 있고 거기서 미주 녹스가 잘 붙는지 테스트

버지니아 세팅하는데 한시간걸림. 23비트 대역만 있으면 댐 90대역. vpn gw까지 리소스만 만드는거 1시간되고 컨피그 시간은 ipsec(하나는 스태틱, 하나는 뉴저지랑 bgp) 양쪾 내부 피어대역 받아서 구성하면됨 대역만 나오면 바로바로 반영 가능함

이전과 차이점이 있나

관계사쪽(구ㅡ주)는 애저쪽으로 가는건 맞는데 녹스관문은 두번짼데 건설은 bgp 안했음 (어렵거나 그런건 아님), 근데 걸리는건 관문을 메인이랑 백업 두개로 하는걸 미주 녹스 대상으로 하는건 처음이다
