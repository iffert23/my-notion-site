데이터암호화

데이터를 저장할 때 암호화 / 데이터가 전송될 때 암호화

저장 중 암호화 : 스토리지, vm, paasdb, aks, 백업몰트, 기타

CMK : 고객이 직접 관리하는 키.

애저에서 항상 사용하는 SSE 사용 → PMK라는 방식으로 키를 관리함

키 관리 : 백업이랑 결이 비슷함. 하나의 서비스 하나의 라이프사이클. 같은 라이프사이클을 가진 리소스가 동일하게 사용할 수 있게 함

DB암호화. TDE 암호화가 걸리게 된다

전송 중 암호화 : L7에서 오프로딩 해서 HTTPS→HTTP하는거, 외부 데이터 들어왔을 때 L7에서 암호화하고 엔드포인트에서 복호화

Azure Policy 연계— 각 리소스 별 보안 전송을 위한 TLS 버전 설정 및 HTTPS 통신을 강제하여 보안을 강화

모니터링, 로깅

인프라 모니터링, 보안 로깅에 대한 모니터링 두 가지로 목적이 나뉨

어떤 리소스의 어떤 항목을 수집할건지, 수집 이후 보관은?, 수집하고 어떻게 시각화? →를 정의

entra id 로그, 구독 활동로그, network(FW, NSG) 로그 → log analytics 저장, sentinel에서 분석, 둘을 연동 / 리소스 로그 → 스토리지(보존, 수명관리정책 할당) ⇒ 를 허브 영역에 저장.

어떤 리소스의 어떤 항목을 볼건지, 선택해서 볼건지 다 볼건지, 저장하면 돈이 나옴..마니… 그러니까 중요한것만 추려서 선별해야함
