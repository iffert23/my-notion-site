DNS

프라이빗 dns 리졸버 → PaaS 형태로 애져에서 매니지먼트 해주는 솔루션

최근에는 많이 씀. 나온지 2~3년

온프렘에 DNS 서버를 두고 포워딩해서 사용하는건 비추

사용이 안되는건 아니지만 관리적인 부분에서는 불편하고 완벽한 DNS 솔루션이라고 하기 어려움 → 가능은 한데 권장은 안한다

한곳에서 DNS를 다 관리하겠다.

VWAN에 이미 프라이빗 리졸버가 있었고 각 리소스에 흩어져있는 DNS를 리졸버로 한 곳으로 모으는 형태로 작업함

옵션에 프라이빗 DNS가 있으면 기본으로 생성됨. 근데 그게 구독별로 DNS가 따로따로임

→ 각 구독마다 DNS가 생겨버리기 때문에 vWAN이 있는 구독에 프라이빗 리졸버 DNS를 만듬 → 하이브리드 환경에서 DNS를 한번에 관리하고 싶어서 사용함

포워딩 룰셋 적용

간단한 테스트 가능

온프렘에서 DNS를 사용하고 있고 애져 통합하면서 DNS를 통일하고 싶고 통일하기 위해서 리졸버 사용함

온프렘 DNS를 slave로, SAP 구독은 Master

vWAN 구독을 제외한 다른 리소스에서는 프라이빗 dns를 생성하지 못하게 하는 폴리시 룰을 만들어둠
